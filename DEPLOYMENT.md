## 🔧 Step-by-Step Deployment

### **1. Set Up MongoDB Atlas**

1. Go to https://mongodb.com/cloud/atlas
2. Create free account (M0 tier - 512MB free)
3. Create cluster → **Name:** `Cluster0`
4. **Database Access:** Create user with password
5. **Network Access:** Allow access from anywhere (`0.0.0.0/0`)
6. Get connection string:
   - Click "Connect" → "Connect your application"
   - Copy URI: `mongodb+srv://username:password@cluster0.xxxxx.mongodb.net/`
   - Add database name: `mongodb+srv://...net/kirti_portfolio`
7. Save this URI for Render environment variable

---

### **2. Deploy Backend to Render.com**

1. Push code to GitHub (see GitHub setup below)
2. Go to https://render.com → Sign up/Login
3. **New → Web Service**
4. Connect your GitHub repository
5. Render auto-detects `render.yaml` ✅
6. **Or manually configure:**
   - **Name:** `kirti-portfolio-backend`
   - **Environment:** `Python 3`
   - **Build Command:** `cd backend && pip install -r requirements.txt`
   - **Start Command:** `cd backend && uvicorn server:app --host 0.0.0.0 --port $PORT`
   - **Plan:** Free
7. Add environment variables from table above
8. Click **"Create Web Service"**
9. Wait for deployment (~5-10 mins)
10. Copy your backend URL (e.g., `https://kirti-portfolio-backend.onrender.com`)
11. Update `BACKEND_URL` environment variable with this URL

---

### **3. Deploy Frontend to Vercel**

1. Go to https://vercel.com → Sign up/Login
2. **New Project**
3. Import your GitHub repository
4. Configure:
   - **Root Directory:** `frontend`
   - **Framework Preset:** Create React App (auto-detected)
   - **Build Command:** `yarn build` (auto-detected)
   - **Output Directory:** `build` (auto-detected)
5. Add environment variable:
   - **Name:** `REACT_APP_BACKEND_URL`
   - **Value:** Your Render backend URL from step 2.10
6. Click **"Deploy"**
7. Wait for deployment (~2-3 mins)
8. Your site is live! 🎉

---

### **4. Update CORS in Render**

After Vercel deployment:

1. Copy your Vercel URL (e.g., `https://kirti-portfolio.vercel.app`)
2. Go back to Render → Your backend service → Environment
3. Update `CORS_ORIGINS` to: `https://kirti-portfolio.vercel.app,https://kirtikilledar.com`
4. Save changes (service will auto-redeploy)

## 🐛 Troubleshooting

### Backend won't start on Render
- Check logs in Render dashboard
- Verify all environment variables are set
- Ensure `MONGODB_URI` is correct

### Frontend can't connect to backend
- Check browser console for CORS errors
- Verify `REACT_APP_BACKEND_URL` in Vercel is correct
- Ensure `CORS_ORIGINS` in Render includes your Vercel URL

### Images not uploading
- Verify Cloudinary credentials in Render
- Check backend logs for Cloudinary errors
- Test Cloudinary credentials at cloudinary.com/console

### MongoDB connection failed
- Verify MongoDB Atlas allows connections from `0.0.0.0/0`
- Check database user has correct permissions
- Ensure password doesn't have special characters (URL-encode if needed)
